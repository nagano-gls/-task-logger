# aki-Cockpit v2.0.0

タスク・スケジュール・出退勤を一元化するパーソナル「コックピット」アプリ。
1ファイルのHTML（`index.html`）で動作し、**すべてのデータはブラウザ内（localStorage）にのみ保存されます**。
リリース履歴は `aki-Cockpit_Release-Notes_v2.0.0.xlsx` を参照。

## データ非公開ポリシー（v2.0.0〜）

このリポジトリはGitHub Pages配信のためPublicですが、**公開されるのはアプリのコード（器）だけ**です。

- タスク・業務記録・メモ・予定・出退勤データは、使用している端末のブラウザ内（localStorage）にのみ保存されます。外部サーバーやこのリポジトリに送信・保存されることはありません
- 公開URLを第三者が開いても、空のアプリが表示されるだけです
- v2.0.0で、予定データをリポジトリにコミットするGitHub Actionsワークフローを全廃しました（データが公開領域に乗る経路をなくすため）
- バックアップはユーザー自身のダウンロード操作（設定タブのエクスポート／退勤時の自動JSONダウンロード）でローカルに保存されます

## 主な機能

- **1日の流れ型UI**: MORNING「今日を計画する」（出勤・今日の予定・タスク追加）→ DAYTIME「今日を実行する」（タスク管理・時間記録）→ NIGHT「今日を振り返る」（所感・メモ）
- **日付＋挨拶ヘッダー**: 設定タブの「表示名」で「こんにちは、○○さん」表示
- **リンクチップ**: 設定タブで登録したリンクをブランド色風グラデーションのチップとして常時表示
- **予定タブ**: .icsファイルを読み込んで今日〜2週間の予定を表示。複数カレンダー（スタジオ予約＋個人予定など）を色分けバッジで混在表示。繰り返し予定・例外日対応
- **freee打刻連携**: 出勤・退勤ボタンでfreee打刻ページを自動で開く（設定でON/OFF）。ヘッダーに「🕐 打刻」ボタン常設
- **PWA/モバイル対応**: スマホは下部ナビゲーション、「ホーム画面に追加」でアプリとして起動

## 予定表の読み込み方法

1. **GMO ONAiRスタジオ予約など、ICS配信URLがある場合**: URLをブラウザで開くと.icsがダウンロードされる → 予定タブの「📂 .ics読み込み」で読み込む（URLをリンク集に登録しておくと楽）
2. **Outlook個人予定**: デスクトップ版Outlookで「ファイル → 予定表の保存」→「その他のオプション」で期間（今後2週間程度）と詳細（**完全な詳細**）を指定して保存 → 同じく「📂 .ics読み込み」
3. 複数の.icsを読み込むと1つの予定表にまとめて色分け表示されます。同名ファイルの再読み込みでその分だけ更新。データが24時間以上古くなると注意表示が出ます

※ 設定タブの「ICS URL直接取得」は、CORS（クロスオリジン）を許可しているカレンダー配信のみ有効です。取得はブラウザ内で完結し、データが外部に出ることはありません。
